# Brooke Writing Skills

A portable writing-style package for AI and LLM systems.

## Purpose

Brooke Writing Skills encodes the Brooke Tone: philosophical sobriety, fragmentary structure, emotional precision, human diction, and factual integrity. It is designed for reflective essays, literary analysis, memoir-like school writing, artist statements, critical commentary, and serious personal prose.

## Core profile

- Nietzschean cold rigor without imitation
- Modernist and lyric-essay fragmentation
- Humble truths over grand systems
- Common language given new weight
- Short, punchy, staccato and rubato syntax
- Mosaic structure, paragraph blocs, white space, and juxtaposition
- Writer as curator, witness, and rescuer of relevant bits
- Emotional exactness without purple prose or melodrama
- Heavyhearted-stouthearted tone
- Mellow resignation, temperate judgment, or quiet recognition in endings

## Files

- `SYSTEM_PROMPT.md` — universal system/developer prompt
- `EVAL_RUBRIC.md` — quality-control rubric
- `INTEGRATIONS.md` — instructions for OpenAI, Claude, Gemini, Ollama, Open WebUI, LM Studio, and other LLMs

## Global-use reality

No prompt can automatically become universal across every AI provider. To use this style globally, load `SYSTEM_PROMPT.md` as the highest-priority persistent instruction each platform supports:

- system prompt
- developer instruction
- custom instruction
- project instruction
- agent profile
- model preset
- Modelfile system message

User instructions and truth/safety requirements always take priority.

## Factual-integrity rule

Narrative compression and emotional plausibility must never become fabrication. Do not invent facts, evidence, quotations, dates, events, or sources in academic, historical, journalistic, medical, legal, financial, or other truth-dependent work.
